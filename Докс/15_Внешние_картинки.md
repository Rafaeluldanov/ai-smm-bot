# Внешние картинки (Этап 9)

Документ описывает backend-логику **поиска, проверки и безопасного использования внешних изображений**. Если у поста/темы нет подходящего собственного approved-медиа, бот может предложить внешнее изображение — но строго как иллюстрацию, с учётом лицензии и без выдачи за свой кейс. Реальные стоки и сеть на этом этапе **не** подключены (fake-провайдер); файлы не скачиваются.

## Зачем нужны внешние картинки

Пример: тема «Шелкография на футболках», своих approved-медиа нет. Бот ищет внешние варианты: фото процесса шелкографии, текстильная печать, изделие без чужого логотипа. Это закрывает пробел в иллюстрациях, пока не сделана своя съёмка.

## Почему нельзя брать любую картинку из интернета

Внешняя картинка:

- **не является нашим кейсом** и не нашей работой;
- может использоваться **только как иллюстрация**;
- требует хранения источника, автора, лицензии и ограничений;
- требует review перед использованием в рекламных/коммерческих публикациях.

Поэтому каждый найденный вариант сохраняется как `ExternalImageCandidate` с полным набором прав и флагов безопасности, а не сразу как готовое медиа.

## review_status

| Статус | Смысл |
|--------|-------|
| `candidate` | найден, не обработан |
| `needs_review` | требует ручной проверки (нужна атрибуция / есть люди / есть логотип) |
| `approved` | одобрен как иллюстрация |
| `rejected` | отклонён (некоммерческая лицензия, небезопасно и т. п.) |
| `converted_to_media_asset` | сконвертирован в `MediaAsset` |

Статус проставляется автоматически при поиске и может меняться вручную через `PATCH /external-images/{id}/review`.

## Лицензии

Хранятся: `license_name`, `license_url`, `commercial_use_allowed`, `modification_allowed`, `attribution_required`, а также флаги `contains_people`, `contains_logo`, `safe_for_business`. Политика (`external_image_license_policy.py`) выводит из них:

- `can_use_organically` — можно ли в обычных публикациях (нужны commercial + safe);
- `can_use_in_ads` — можно ли в рекламе (commercial + safe, без чужого логотипа и людей);
- `required_attribution` — текст указания автора (если `attribution_required`);
- `forbidden_usage` — список запрещённых способов (всегда включает `claim_as_own_case`).

Фильтры поиска по умолчанию: `require_commercial_use`, `require_no_logo`, `require_safe_for_business` — небезопасные/некоммерческие/с логотипом варианты отсеиваются.

## Внешнее нельзя выдавать за свой кейс

`can_claim_as_own_case` **всегда `false`**. `forbidden_usage` всегда содержит `claim_as_own_case` и `portfolio`. При конвертации в `MediaAsset` в результат добавляется предупреждение «Внешнее изображение нельзя использовать как наш кейс/портфолио». Это правило закреплено тестами.

## Как candidate превращается в MediaAsset

`convert_candidate_to_media_asset`:

- **нельзя** конвертировать `rejected`, некоммерческое или небезопасное (→ 409 `ExternalImageConversionError`);
- создаётся `MediaAsset` с `source_type=external_stock`, лицензией `external_needs_review` (если нужна атрибуция) или `commercial_use_required` — **никогда** `company_owned`;
- `yandex_disk_path` = переданный путь или `external://{provider}/{candidate_id}`;
- статус медиа — `needs_license_review` (по умолчанию);
- кандидат получает `media_asset_id` и статус `converted_to_media_asset`;
- в предупреждениях — запрет на «свой кейс» и (при необходимости) требование указать автора.

## API и команды

REST API `/external-images` (`backend/app/api/external_images.py`):

- `GET /external-images?project_id=&topic_id=&post_id=&provider=&review_status=` — список;
- `GET /external-images/{candidate_id}` — кандидат (404);
- `POST /external-images/search` — поиск (404 — нет проекта/темы/поста);
- `POST /external-images/search/post/{post_id}` и `POST /external-images/search/topic/{topic_id}` — поиск под пост/тему;
- `GET /external-images/{candidate_id}/safety` — оценка безопасности;
- `PATCH /external-images/{candidate_id}/review` — смена статуса (422 — неизвестный статус);
- `POST /external-images/{candidate_id}/convert-to-media` — конвертация (409 — нельзя).

CLI:

```bash
make search-external-images project_slug=teeon query="шелкография"
make convert-external-image candidate_id=1
```

## Ограничения Этапа 9

- **только fake-провайдер** — реальные стоки (Unsplash/Pexels/Google) не подключены;
- **изображения не скачиваются**, сеть не вызывается;
- Instagram и AI не подключены;
- внешнее изображение **никогда** не выдаётся за наш кейс/портфолио.
