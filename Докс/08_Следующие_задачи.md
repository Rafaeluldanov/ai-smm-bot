
# Следующие задачи (Production hardening / MVP-релиз)

Все этапы разработки **0–10 завершены**. Реализован сквозной автономный pipeline: выбор тем → генерация постов → подбор медиа → внешние картинки → согласование → планирование → публикация → аналитика → отчёт, с safety-guardrails и режимами (`dry_run`/`semi_auto`/`auto_generate`/`auto_schedule`/`auto_publish`). Всё работает на fake-провайдерах без реальной сети и AI.

Следующий блок работ — не новый этап разработки, а **подготовка к боевому запуску (MVP)**.

## Production hardening / MVP-релиз

Чтобы запустить бота в бою, нужно (вне текущего поэтапного цикла):

- [ ] Реальные токены (`TELEGRAM_BOT_TOKEN`, `VK_ACCESS_TOKEN`, `YANDEX_DISK_TOKEN`) в защищённом хранилище секретов.
- [ ] Реальные Telegram/VK каналы и группы; включение живого режима в публикующих клиентах.
- [ ] Мониторинг и алерты (ошибки публикации, упавшие прогоны, нехватка медиа).
- [ ] Структурированное логирование и трассировка сложных действий.
- [ ] Управление секретами и ротация токенов.
- [ ] Резервное копирование БД (backup/restore).
- [ ] UI/админка для согласования постов и просмотра прогонов/аналитики.
- [ ] Ручное QA на реальных проектах перед запуском.

## Критерий готовности MVP

- бот ведёт реальные соцсети проектов под контролем человека;
- секреты, мониторинг и бэкапы настроены;
- есть админ-интерфейс для ревью;
- проведено ручное QA;
- `make check` проходит.
