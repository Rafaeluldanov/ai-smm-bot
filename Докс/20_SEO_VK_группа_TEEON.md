# SEO-заполнение и SEO-публикации VK-группы TEEON

Документ описывает системный модуль SEO для VK-группы **TEEON** (`group_id 240102732`)
и архитектуру под вторую группу — **Фабрика сувениров** (`fabric-souvenirs`).

Модуль решает две задачи:

1. **SEO-заполнение группы** — описание, статус, закреплённый пост, услуги, меню,
   хэштеги, ссылки (превью, без реальных изменений VK).
2. **SEO-публикации** — контент-план с релевантными ссылками на страницы сайта
   [teeon.ru](https://teeon.ru), подбором собственного медиа по тегам и CTA.

> ⚠️ **Безопасность.** На этом этапе реальные публикации и реальные изменения
> оформления VK-группы **не выполняются**. Всё работает как preview / dry-run.
> `VK_LIVE_PUBLISHING_ENABLED` и `VK_GROUP_SETUP_LIVE_ENABLED` по умолчанию `false`.

## Откуда берутся данные

| Источник | Что даёт | Где в коде |
|----------|----------|------------|
| Сайт teeon.ru (каталог) | страницы `/catalog/...` под изделия | `seo_content_sources._TEEON_CATALOG_PAGES` |
| Сайт teeon.ru (нанесения) | страницы `/branding/...` под технологии | `seo_content_sources._TEEON_BRANDING_PAGES` |
| Позиционирование, доверие, контакты | описание группы, закреп | `seo_content_sources` (`positioning`, `trust_facts`, `Contacts`) |
| Seed-ядро Яндекса | SEO-запросы по кластерам | `teeon_seo_queries` |
| Контентный вектор | приоритеты изделий/технологий, контент-микс | `seo_content_sources._TEEON_CONTENT_VECTOR` |
| MediaAsset / Яндекс Диск | фото/видео по тегам (свои, company_owned) | `seo_media_selection_service` |

Все данные **детерминированы и оффлайн**: без сети, без AI, без обращений к БД
(медиа-подбор — единственное место, где нужна БД, и он опционален).

## SEO-профиль проекта

`ProjectSeoProfile` (`app/services/seo_content_sources.py`) — единый источник:

- `brand_name`, `site_url`, `contacts` (телефон `+7 (495) 152-37-45`,
  email `teeon@upgifts.ru`, город Москва, график Пн–Пт 10:00–19:00);
- `catalog_pages` / `branding_pages` / `other_pages` — страницы сайта с тегами
  продуктов/технологий;
- `seo_queries` — seed-ядро;
- `content_vector` — приоритеты и тон;
- `vk_group_id`, `vk_screen_name`.

```python
from app.services.seo_content_sources import get_project_seo_profile
profile = get_project_seo_profile("teeon")
```

## SEO-ядро (seed-запросы)

`teeon_seo_queries.build_teeon_seo_queries()` возвращает запросы, сгруппированные
по кластерам (`футболки / майки`, `худи / толстовки / свитшоты`, `лонгсливы`,
`кепки / бейсболки`, `жилетки`, `куртки`, `дождевики`, `мерч под ключ`,
`корпоративная одежда`, технологии, `производство / пошив / цех`, `опт / B2B`).

Для каждого запроса выводятся `product`, `technology`, `intent`
(`commercial/price/process/brand/informational`) и `priority` из частотности.
**Нулевые запросы сохранены** — они нужны как long-tail и заготовки под будущие
разделы/посты.

## Ссылки на сайт

`site_link_selection_service.select_site_link(...)` выбирает **одну** релевантную
страницу под тему поста (тема, заголовок, теги медиа, продукты, технологии,
SEO-кластер). Правила:

- названная **технология** → страница нанесения (DTF → `/branding/dtf-pechat`,
  вышивка → `/branding/vyshivka`, гравировка → `/branding/gravirovka`,
  шелкография → `/branding/shelkografiya`, бирки → `/branding/birki`,
  шевроны → `/branding/shevrony`);
- **УФ-печать** и **термотрансфер** отдельной страницы не имеют → корневая
  `/branding`;
- **продукт** → страница каталога (футболки → `/catalog/futbolki`,
  худи → `/catalog/hudi`, кепки → `/catalog/kepki` и т. д.);
- общая тема «мерч под ключ» → `/korporativnyy-merch`;
- кейсы/портфолио → `/portfolio`; если точной страницы нет → главная.

`PostGenerationService` добавляет **одну** ссылку в конец VK/Telegram-текста:
`Подробнее и расчёт тиража: https://teeon.ru/catalog/futbolki` (без спама).

## Контент-план

`SeoContentPlanService.build_plan(project_slug, days=30)` строит план на N дней:
дата/день, рубрика, тема, SEO-запрос, продукт/технология, ссылка на сайт,
рекомендуемый медиа-тег, CTA. Рубрики распределяются по контент-миксу
(**товары 30% / технологии 30% / производство 20% / кейсы 10% / FAQ 10%**),
технологические дни покрывают приоритеты DTF/вышивка/гравировка/УФ-печать и др.

```bash
make seo-content-plan project_slug=teeon days=30
PYTHONPATH=backend .venv/bin/python -m app.scripts.generate_seo_content_plan \
    --project-slug teeon --days 30
```

Флаг `--with-media` дополнительно подбирает собственное approved-медиа из БД и
показывает `preferred_media_path` улучшенной копии (если она одобрена).

## Подбор медиа

`SeoMediaSelectionService` берёт только собственные медиа проекта:
`source_type=internal`, `license_type=company_owned`, `status ∈ {approved,
approved_video}`; при наличии своих фото исключает `external_reference`;
сопоставляет теги продуктов/технологий. Если есть одобренный enhanced-вариант —
возвращает `preferred_media_path`.

**Изоляция медиа между проектами** (`project_media_paths`): TEEON берёт медиа
только из папки «Тион»; `fabric-souvenirs` — из «Тион» **и** «Фабрика сувениров».
TEEON никогда не использует медиа из папки «Фабрика сувениров».

## Как preview-заполнить VK-группу

`vk_group_seo_setup_service` собирает превью **без единого вызова VK API**:

```bash
make preview-vk-seo project_slug=teeon
PYTHONPATH=backend .venv/bin/python -m app.scripts.preview_vk_group_seo_setup \
    --project-slug teeon
```

Превью содержит: название, статус, короткое и полное описание, закреплённый пост,
услуги, SEO-хэштеги, ссылки, рубрики, меню и предупреждения.

**Статус группы:** `Корпоративный мерч и промо-одежда на заказ | Пошив и нанесение
логотипа`.

**SEO-хэштеги:** `#мерч #корпоративныймерч #промоодежда #футболкислоготипом
#худислоготипом #пошивфутболок #пошивхуди #DTFпечать #вышивка #гравировка
#УФпечать #шелкография #корпоративнаяодежда #пошивмерча #мерчмосква
#производствомерча #TEEON`.

## Live-изменения выключены по умолчанию

`apply_vk_group_setup(project_slug, dry_run=True)`:

- `dry_run=True` (по умолчанию) → возвращает план действий, **ничего не меняет**;
- `dry_run=False` **без** `VK_GROUP_SETUP_LIVE_ENABLED=true` → блокируется
  (`VkGroupSetupLiveDisabledError`, API → 403);
- проект не в `VK_GROUP_SETUP_ALLOWED_PROJECTS` → блокируется (403);
- даже при включённом флаге реальное оформление на этом этапе **не отправляется**
  (safety): `applied=False` с предупреждением.

Настройки (`.env`):

```dotenv
VK_GROUP_SETUP_LIVE_ENABLED=false
VK_GROUP_SETUP_ALLOWED_PROJECTS=teeon,fabric-souvenirs
```

## API

| Метод | Путь | Назначение |
|-------|------|-----------|
| GET | `/seo/project/{slug}/profile` | SEO-профиль проекта |
| GET | `/seo/project/{slug}/vk-group-preview` | превью SEO-заполнения группы |
| GET | `/seo/project/{slug}/content-plan?days=30` | контент-план на N дней |
| POST | `/seo/project/{slug}/vk-group-apply` | apply (dry_run=true по умолчанию; live → 403) |

Эндпоинты профиля/превью/плана не требуют БД и работают, даже когда PostgreSQL
остановлен.

## Контентный вектор TEEON

**Приоритет изделий:** футболки 100, худи 95, свитшоты 80, лонгсливы 80, кепки 75,
жилетки 70, куртки 65, дождевики 60, сумки 55.

**Приоритет технологий:** DTF 100, вышивка 95, гравировка 90, УФ-печать 90,
шелкография 85, DTG 75, термотрансфер 70, шевроны 65, бирки 60. *УФ-печать
заложена приоритетной по требованию владельца, даже если на сайте представлена
слабо.*

**Тон:** B2B, экспертный, уверенный, без хайпа; акцент на качество, сроки, тиражи,
производство, договор, расчёт стоимости.

**Запрещено:** выдавать внешние картинки за свои кейсы; обещать невозможные сроки;
искажать цвет/фактуру; публиковать без review; включать live-publishing по
умолчанию.

Дефолтный бизнес-вектор публикаций (`get_default_publication_vector("teeon")`)
подставляется в автономный прогон флагом `--use-default-publication-vector`
(если пользователь не задал `--business-priority` вручную — ручные приоритеты
всегда сильнее пресета).

## Вторая группа — Фабрика сувениров

Архитектура заложена: отдельный `ProjectSeoProfile` (`fabric-souvenirs`),
плейсхолдер VK-конфига (`vk_group_id=None`, сайт не задан), свой контент-вектор
(кружки, ручки, текстиль, пакеты, подарки; УФ-печать/шелкография/тампопечать/
гравировка) и своё правило доступа к медиа (может брать из папок «Тион» **и**
«Фабрика сувениров»). Полный контент и публикации для сувенирки на этом этапе
**не** реализуются.
