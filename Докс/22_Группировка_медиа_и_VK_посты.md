# 22. Группировка медиа и VK-посты с несколькими фото (v0.1.14)

Модуль превращает синхронизированные и протегированные собственные медиа проекта
в пост с **группой фото**, публикуемый одним VK-постом (несколько вложений).
Всё офлайн-детерминированно; реальная публикация — только вручную и только под
флагом.

## Схема сценария

```
sync (Яндекс Диск)  →  tag (теги медиа)  →  group (похожие медиа)
        →  create post (текст + SEO-ссылка + группа медиа)
        →  dry-run preview (что уйдёт, без сети)
        →  publish (вручную, VK_LIVE_PUBLISHING_ENABLED=true)
```

1. **sync** — `make sync-public-media project_slug=teeon` (публичная папка Диска).
2. **tag** — теги берутся из `MediaAsset.tags`; если пусто/слабо —
   `retag-media` (`MediaAnalysisService` / `MediaTaggingService`). Для видео на
   этом этапе кадры не извлекаются — тегируем по имени файла/пути/уже
   существующим тегам (frame extraction — TODO следующего этапа).
3. **group** — `MediaGroupingService.group_project_media` объединяет похожие
   медиа. Приоритет группировки: **products → technologies → topics →
   categories → details**. Примеры групп: футболка, худи, DTF, шелкография,
   вышивка, швейный цех.
4. **create post** — `create_post_from_media_group` собирает пост со ссылкой на
   `teeon.ru` (через `SiteLinkSelectionService`) и группой медиа; статус по
   умолчанию `needs_review`.
5. **dry-run** — `publish_post --dry-run` показывает `media_kind`, `media_count`,
   `preferred_media_path`, `would_attach_media`, предупреждения. Ничего не шлёт.
6. **publish** — ручная публикация одного поста (см. ниже).

## Строгий отбор медиа

В группу берутся только собственные пригодные к публикации активы:

- `source_type = internal` и `license_type = company_owned`;
- `status` из `{approved, approved_video}`;
- исключаются `external_reference` / `external_stock` (по категориям тегов и
  источнику);
- исключаются `rejected`, `new`, `needs_reshoot`, `needs_license_review`.

Если у актива есть одобренная улучшенная копия (approved enhanced variant), для
загрузки предпочитается её путь (`media_path`); оригинал не перезаписывается.
Если оригинал `HEIC/HEIF` и готовой JPG-копии нет — для VK-загрузки выполняется
временная **in-memory** конвертация в JPEG (Pillow / `ImageEnhancementProcessor`),
оригинал остаётся нетронутым.

## Фото и видео

- **Фото поддерживаются**: несколько фото уходят одним VK-постом; вложения —
  `photo{owner_id}_{id}` через запятую. Лимит по умолчанию — **5 фото** в посте
  (`VK_MEDIA_GROUP_MAX_PHOTOS`); размер группы дополнительно ограничивается
  CLI-флагом `--limit-media`.
- **Видео пока не загружается**: попадает в группу и учитывается в счётчиках, но
  при публикации пропускается с предупреждением
  `VK video upload is not implemented; video skipped`. Пост уходит с фото и
  текстом.

## Безопасный фолбэк группового токена

Групповой токен VK может не иметь прав на `photos.getWallUploadServer` /
`photos.saveWallPhoto` (VK `error_code=27`). Тогда пост **не падает**: публикуется
text-only, а в `raw` попадают:

- `media_upload_skipped = true`;
- `media_upload_error_code = 27`;
- `media_warnings` с пояснением.

Любая другая ошибка VK API — это `PublishError` (пост не публикуется).

## generation_notes

Группа медиа хранится в `Post.generation_notes` (JSON):

```json
{
  "media_group_key": "футболка",
  "media_group_tags": ["футболка"],
  "media_asset_ids": [12, 13, 14],
  "media_files": [{"id": 12, "file_name": "...", "yandex_disk_path": "...",
                   "media_path": "...", "media_source": "enhanced_variant",
                   "media_kind": "image"}],
  "media_count": 3,
  "image_count": 3,
  "video_count": 0,
  "selected_for_vk_upload": true,
  "warnings": []
}
```

`PostPublicationService.build_publish_request` читает `media_asset_ids`: если их
больше одного — собирает несколько вложений (`media_items`); одиночный
`media_asset_id` продолжает работать по-старому.

## CLI

Превью групп (ничего не создаёт):

```bash
PYTHONPATH=backend .venv/bin/python -m app.scripts.preview_media_groups \
  --project-slug teeon --tag футболка --max-groups 10 --limit-media 5 --include-videos
# или: make media-groups project_slug=teeon tag=футболка
```

Создание поста из группы (needs_review):

```bash
PYTHONPATH=backend .venv/bin/python -m app.scripts.create_media_group_post \
  --project-slug teeon --tag футболка --limit-media 5 --status needs_review
# или: make media-group-post project_slug=teeon tag=футболка
```

## Ручная публикация одного поста

Только вручную и только при `VK_LIVE_PUBLISHING_ENABLED=true` (флаг в `.env`):

```bash
# 1. Согласование
python -m app.scripts.review_post --post-id <ID> --action submit
python -m app.scripts.review_post --post-id <ID> --action approve
# 2. Планирование
python -m app.scripts.schedule_post --post-id <ID>
# 3. Dry-run превью (без сети)
python -m app.scripts.publish_post --post-id <ID> --dry-run
# 4. Живая публикация ОДНОГО поста (только вручную)
python -m app.scripts.publish_post --post-id <ID> --platform vk
```

> `publish-due` (планировщик созревших публикаций) на этом этапе **не
> использовать** — публикуем строго по одному посту вручную.
