
# Логика выбора тем

Этот документ описывает Этап 4 — механизм, с помощью которого AI-SMM-бот самостоятельно выбирает темы для продвижения проектов.

На этом этапе бот ещё не пишет полноценные посты и не публикует их в соцсети. Его задача — выбрать темы, рассчитать приоритет и сформировать недельный контент-план.

## Главный принцип

Человек задаёт не конкретные темы постов, а стратегические направления.

Пример запроса:

```json
{
  "business_priorities": {
    "футболки": 100,
    "худи": 80,
    "шелкография": 90
  },
  "weeks": 1,
  "posts_per_week": 3
}
```

Бот сам выбирает конкретные темы:

- «Футболки с логотипом на заказ».
- «Футболки для корпоративного мерча».
- «Шелкография на футболках».
- «Худи для команды».
- «Худи с жаккардовыми элементами».

## Сигналы выбора

| Сигнал | Значение |
|---|---|
| `business_priority` | Насколько направление важно бизнесу сейчас |
| `media_readiness_score` | Есть ли готовые медиа под тему |
| `search_demand_score` | Условная частотность/спрос |
| `commercial_intent_score` | Коммерческая ценность темы |
| `trend_score` | Условная актуальность темы |
| `seasonality_score` | Сезонность |
| `competition_score` | Конкурентность темы |

Реальные Wordstat, Google Trends и внешние API на Этапе 4 не подключены. Используется статический `StaticMarketSignalProvider`.

## Формула scoring

Итоговый `priority_score` считается по шкале 0–100.

```text
priority_score =
  search_demand_score * 25
+ commercial_intent_score * 25
+ trend_score * 10
+ seasonality_score * 10
+ media_readiness_score * 15
+ business_priority_score * 20
- competition_score * 5
```

Результат ограничивается диапазоном 0–100.

## Business priority

`business_priority` — ручной бизнес-сигнал.

Пример:

```json
{
  "футболки": 100,
  "худи": 80,
  "шелкография": 90
}
```

Если тема относится к кластеру `футболки`, она получает высокий вклад в скоринг.

## Media readiness score

`media_readiness_score` показывает, насколько тема готова к публикации с точки зрения визуалов.

| Ситуация | Score |
|---|---:|
| Есть `approved` или `approved_video` медиа | 1.0 |
| Есть только `new` медиа | 0.6 |
| Есть только `needs_reshoot` | 0.3 |
| Медиа нет | 0.0 |

Если медиа нет, в контент-плане появляется `needs_media=true` и `suggested_media_query`.

## Explanation

Каждая выбранная тема получает объяснение.

Пример:

```text
Тема выбрана: высокий бизнес-приоритет направления 'футболки';
высокая коммерческая релевантность; высокий поисковый спрос;
медиа по теме пока нет (нужна досъёмка).
```

## Weekly content plan

Для `posts_per_week = 3` используются дни:

| Slot | День |
|---:|---|
| 1 | Понедельник |
| 2 | Среда |
| 3 | Пятница |

## Ограничения Этапа 4

На Этапе 4:

- нет реального Wordstat;
- нет реального Google Trends;
- нет AI-анализа рынка;
- нет генерации текстов постов;
- нет автопостинга;
- нет рекламных кабинетов.

Этап 4 отвечает только за выбор тем и формирование контент-плана.
