# Словарь медиа-тегов

Документ описывает систему тегирования медиа-активов, реализованную на **Этапе 3 «Анализ медиа»** (поверх Этапов 0, 1, 2).

На этом этапе теги строятся **по имени файла и пути на Яндекс Диске** — реальный AI-анализ изображений (vision) пока **не подключён**. В коде есть только задел на будущее (`backend/app/ai/vision.py`: `StubVisionAnalyzer` всегда бросает `VisionAnalysisNotConfiguredError`). Когда vision будет подключён, он дополнит, а не заменит, текущую логику по именам файлов.

Источники истины:
- Словари терминов и функции разбора: `backend/app/services/media_taxonomy.py`
- Сервис тегирования: `backend/app/services/media_tagging_service.py`
- Сервис анализа и рекомендаций: `backend/app/services/media_analysis_service.py`

---

## 1. Группы тегов

Каждый медиа-актив получает структуру `tags` со следующими группами. Значения — это нормализованные термины из словарей таксономии.

### products — изделия

Что изображено как продукт. Базовая и самая «весомая» группа.

Примеры: `футболка`, `худи`, `свитшот`, `лонгслив`, `поло`, `шоппер`, `шопер`, `кепка`, `бейсболка`, `кружка`, `ручка`, `пакет`, `ежедневник`, `блокнот`, `термокружка`, `бутылка`, `значок`, `брелок`.

Словарь: `PRODUCT_TERMS`.

### technologies — технологии нанесения / производства

Как нанесён принт или декор. Вторая по весу группа.

Примеры: `шелкография`, `dtf`, `dtg`, `вышивка`, `сублимация`, `уф-печать` (`уф печать`), `тампопечать`, `гравировка`, `тиснение`, `термоперенос`, `шильд`.

Словарь: `TECHNOLOGY_TERMS`. Короткие коды (`dtf`, `dtg`, `uv` и т. п.) сопоставляются только точно (см. раздел 6).

### details — детали изделия

Конструктивные и оформительские детали. По правилам Этапа 3 сюда отнесены **жаккард, бирка, шеврон** (это детали, а не технологии).

Примеры: `жаккард`, `бирка`, `шеврон`, `карман`, `капюшон`, `молния`, `манжет`, `воротник`, `нашивка`, `лейбл`, `кант`.

Словарь: `DETAIL_TERMS`.

### materials — материалы

Из чего сделано изделие.

Примеры: `хлопок`, `футер`, `трикотаж`, `флис`, `керамика`, `металл`, `пластик`, `стекло`, `кожа`, `эко-кожа`, `полиэстер`.

Словарь: `MATERIAL_TERMS`.

### colors — цвета

Примеры: `белый`, `чёрный`, `серый`, `синий`, `красный`, `зелёный`, `бежевый`, `жёлтый`, `розовый`.

Словарь: `COLOR_TERMS`.

### categories — категории актива

Высокоуровневые категории, выводимые из остальных групп и метаданных. Значения:

| Категория | Когда присваивается |
|---|---|
| `apparel` | есть изделие одежды (футболка, худи и т. п.) |
| `souvenirs` | есть сувенирный продукт (кружка, ручка, ежедневник и т. п.) |
| `branding` | присутствует хотя бы одна технология нанесения |
| `production` | есть слова процесса/цеха (`цех`, `производство`, `станок`, `процесс`) |
| `packaging` | упаковка (`упаковка`, `коробка`) |
| `event_merch` | мероприятие/выставка (`мероприятие`, `выставка`, `конференция`, `event`) |
| `external_reference` | `source_type == external_stock` или путь содержит `04_Внешние_картинки_из_интернета` |

### use_cases — сценарии использования

Примеры: `корпоратив`, `мерч`, `подарок`, `промо`, `униформа`, `спорт`, `повседневный`.

Словарь: `USE_CASE_TERMS`.

### audiences — аудитории

Примеры: `b2b`, `сотрудники`, `клиенты`, `дети`, `спортсмены`.

Словарь: `AUDIENCE_TERMS`. Короткий код `b2b` сопоставляется только точно.

## 2. Производные поля

Эти поля вычисляются на основе найденных тегов.

- **topics** — темы для контент-планирования. Строятся функцией `build_topics_from_tags(tags, project_slug)` из комбинаций продуктов, технологий и деталей с учётом проекта. Используются в `suggest_content_topics`.
- **seo_keywords** — поисковые ключевые слова. Строятся функцией `build_seo_keywords_from_tags(tags, project_slug)`. Используются в `suggest_seo_keywords`.
- **matched_terms** — список «сырых» совпадений: какие токены имени файла к каким терминам словаря были привязаны. Полезно для отладки и проверки качества разбора.

---

## 3. Правила confidence

`confidence` — оценка уверенности тегирования от `0.0` до `1.0`, вычисляется в `calculate_tag_confidence(tags)`.

Вклад групп (суммируется, затем обрезается по максимуму `1.0`):

| Группа (есть хотя бы одно значение) | Вклад |
|---|---|
| products | **+0.45** |
| technologies | **+0.30** |
| details | **+0.10** |
| materials | **+0.05** |
| colors | **+0.05** |
| use_cases | **+0.05** |

Максимум — `1.0` (даже если сумма вкладов больше).

Пример: есть продукт + технология + деталь + цвет → `0.45 + 0.30 + 0.10 + 0.05 = 0.90`.

---

## 4. Правила needs_review и review_reasons

`needs_review` (bool) выставляется в `true`, если выполняется **хотя бы одно** из условий. Каждое сработавшее условие добавляет причину в `review_reasons` (list):

| Условие | Смысл |
|---|---|
| `confidence < 0.35` | слишком мало уверенных тегов |
| нет ни `products`, ни `technologies` | непонятно, что на медиа |
| `source_type == external_stock` | внешний сток требует проверки лицензии/уместности |
| путь содержит `04_Внешние_картинки_из_интернета` | внешняя картинка из интернета |

Если ни одно условие не сработало — `needs_review = false`, `review_reasons` пуст.

---

## 5. Нормализация имени файла

Перед сопоставлением имя файла проходит через `normalize_text`:

1. приведение к **нижнему регистру**;
2. замена **ё → е**;
3. **удаление расширения файла** (`.jpg`, `.png` и т. п.);
4. замена разделителей `_`, `-`, `.` на **пробелы**.

Пример: `Худи_с-карманом.JPG` → `худи с карманом`.

---

## 6. Морфологическое сопоставление

Токены нормализованного имени сопоставляются с терминами словарей по правилам (в `extract_keywords_by_taxonomy`):

- **Точное совпадение токена** — токен равен термину.
- **По префиксу** — термин является префиксом токена: `жаккардами` → `жаккард`.
- **По общему префиксу с расхождением в хвосте ≤2 символов** — `шелкографией` → `шелкография`.
- **Многословные термины** — ищутся как **подстрока** во всей нормализованной строке (например, `уф печать`).
- **Короткие коды** (`dtf`, `dtg`, `uv`, `hr`, `b2b`) — **только точное совпадение** (чтобы не ловить случайные подстроки).

## 7. Примеры разбора файлов

> Цвета/категории зависят от метаданных (`source_type`, путь); ниже — ожидаемые теги по имени файла для внутреннего медиа.

### «Худи с карманом с шелкографией и жаккардами.jpg»

- products: `худи`
- technologies: `шелкография` (из «шелкографией» по общему префиксу)
- details: `карман`, `жаккард` (из «жаккардами» по префиксу)
- colors: —
- categories: `apparel` (есть одежда), `branding` (есть технология)
- confidence ≈ `0.45 + 0.30 + 0.10 = 0.85`

### «Футболка белая DTF тираж 500.png»

- products: `футболка`
- technologies: `dtf` (точное совпадение короткого кода)
- details: —
- colors: `белый` (из «белая»)
- categories: `apparel`, `branding`
- confidence ≈ `0.45 + 0.30 + 0.05 = 0.80`

### «Кружка с УФ печатью логотипа.jpg»

- products: `кружка`
- technologies: `уф-печать` (многословный термин «уф печать» как подстрока)
- details: —
- colors: —
- categories: `souvenirs` (сувенирный продукт), `branding`
- confidence ≈ `0.45 + 0.30 = 0.75`

### «Ежедневник с тиснением и гравировкой.jpg»

- products: `ежедневник`
- technologies: `тиснение` (из «тиснением»), `гравировка` (из «гравировкой»)
- details: —
- colors: —
- categories: `souvenirs`, `branding`
- confidence ≈ `0.45 + 0.30 = 0.75`

## 8. Примеры по проектам

### TEEON (одежда / мерч)

Профиль проекта — одежда и мерч с нанесением. Типичные теги:

- products: `футболка`, `худи`, `свитшот`, `лонгслив`, `поло`, `шоппер`, `кепка`
- technologies: `шелкография`, `dtf`, `dtg`, `вышивка`, `сублимация`
- details: `карман`, `капюшон`, `жаккард`, `бирка`, `шеврон`, `молния`
- materials: `хлопок`, `футер`, `трикотаж`, `флис`
- categories: преимущественно `apparel` + `branding`

Целевые темы для досъёмки (`suggest_shooting_tasks`): `футболка`, `худи`, `шоппер`, `шелкография`, `dtf`, `вышивка`, `жаккард`. Если одобренных медиа по теме меньше 2 — формируется задача с папкой `06_Нужно_переснять` и списком кадров.

Пример файла: `Свитшот серый с вышивкой на груди.jpg`
- products: `свитшот`; technologies: `вышивка`; colors: `серый`; categories: `apparel`, `branding`.

### Фабрика сувениров (сувенирка)

Профиль проекта — сувенирная продукция с нанесением. Типичные теги:

- products: `кружка`, `ручка`, `пакет`, `ежедневник`, `блокнот`, `термокружка`, `брелок`, `значок`
- technologies: `уф-печать`, `тампопечать`, `гравировка`, `шелкография`, `тиснение`, `сублимация`
- materials: `керамика`, `металл`, `пластик`, `стекло`
- categories: преимущественно `souvenirs` + `branding`

Целевые темы для досъёмки: `кружка`, `ручка`, `пакет`, `уф-печать`, `тампопечать`, `гравировка`, `шелкография`. При нехватке одобренных медиа (<2) по теме — задача на досъёмку в `06_Нужно_переснять`.

Пример файла: `Ручка металлическая с гравировкой логотипа.jpg`
- products: `ручка`; technologies: `гравировка`; materials: `металл`; categories: `souvenirs`, `branding`.

---

## 9. Важное ограничение

**Vision-анализ изображений пока не подключён.** Все теги на Этапе 3 строятся **по имени файла и пути** на Яндекс Диске. В коде присутствует только интерфейс на будущее (`BaseVisionAnalyzer`, `VisionAnalysisResult`, `StubVisionAnalyzer`), который сейчас не выполняет реального анализа. Поэтому качество тегов напрямую зависит от осмысленности имён файлов — рекомендуется называть файлы по содержанию (продукт, технология, цвет, деталь).
